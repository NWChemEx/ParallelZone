Enables implicit conversion from BinaryView to ConstBinaryView.

This ctor allows BinaryView instances to be implicitly converted to ConstBinaryView instances. This parallels the usual C++ practice of a type T being implicitly convertible to type const T.

template<typename T >
T from_binary_view ( const ConstBinaryView &  view)
related

Wraps the process of going from a BinaryView back to an object.

This function is a convenience function for creating an object from a view of contiguous binary data. The ConstBinaryView is assumed to point to the serialized form of an object of type T if NeedsSerialized<T>::value is true. Otherwise it is assumed that T can be created by copying the data out of view.

Note
When T does not need to be serialized, you often do not need to call this function. Instead consider having the BinaryView directly alias the memory where you want to create the T object.
Template Parameters
TThe type we want to convert the binary data to. T is an explicit template type parameter and must be specified by the user. T should be an unqualified type (i.e., no const, references, or the like). If needs_serialized<T> is false we assume that T has a range ctor that can be used to fill in a new T instance.
Parameters
[in]viewAn alias of contiguous binary data, from which we will make the object.
Returns
An instance of type T initialized from the binary data aliased by view.

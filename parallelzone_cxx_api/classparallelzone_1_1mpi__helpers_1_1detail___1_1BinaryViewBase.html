Detailed Description

template<bool IsConst>
class parallelzone::mpi_helpers::detail_::BinaryViewBase< IsConst >

Guts of BinaryView templated on const vs. non-const.

Ultimately we need two BinaryView types a non-const and a const type. The implementations of the two BinaryView types really only differ in the member types. This means we really could implement these two types as a single templated class and use template meta-programming to correctly set the types; however, for a more user-friendly experience, we don't want BinaryView to be templated. Hence the the BinaryViewBase class, which is templated on the const-ness. BinaryView and ConstBinaryView are more or less strong types derived from the appropriate specialization of BinaryViewBase.

Note
BinaryViewBase is non-owning of the bytes it is a view of. It is the users responsibility to ensure the underlying memory remains valid.
Template Parameters
IsConstUsed to toggle whether this is implementing BinaryView or ConstBinaryView respectively.

Creates a view of the bytes pointed to by p.

This ctor takes a pointer p to the beginning of a buffer. The buffer contains n elements of type T. The resulting BinaryViewBase instance will be a view of this buffer, but as bytes of raw memory.

template<bool IsConst>
size_type parallelzone::mpi_helpers::detail_::BinaryViewBase< IsConst >::size ( ) const
inlinenoexcept

The size of the binary buffer, in bytes.

BinaryViews offer a view of contiguous binary data. This method is used to determine how many bytes the contiguous binary data holds. This method always returns the number of bytes regardless of what type the objects actually are. For example the following assert should be true:

BinaryViewBase bb(double{1.1}, 1);
assert(bb == 1 * sizeof(double));

Detailed Description

Infrastructure for type-erasing a task.

A lot of ParallelZone revolves around scheduling, running, and measuring "tasks". To PZ, a task is nothing more than executing a callable. The callable may or may not return something and it may have side-effects (e.g., logging). The TaskWrapper class provides a generic type-erased interface for interacting with the task.

// Say you want to call function foo with arguments arg0, arg1. Then the
// usage is:
auto [task, unwrapper] = make_task(foo, arg0, arg1);
// Run the task
auto result = task();
// Get the result
auto unwrapped_result = unwrapper(result);

inline

Runs the task held by *this.

TaskWrapper objects allow lazy execution the wrapped task. Calling this method will execute the task. Generally speaking, tasks are single use, so this function should only be called once. The TaskWrapper class does NOT enforce this because without knowledge of the wrapped function's implementation it can not know for sure whether this is the case or not.

Returns
A type erased return. If the wrapped function does not return, the resulting object should NOT be unwrapped.
Exceptions
???Throws if the wrapped function throws. Same throw guarantee.

Creates a function which can unwrap a type-erased result.

Template Parameters

This function creates a callable that takes an rvalue reference to a type_erased_return_type object and returns the object that was inside the object. Note that the return from the callable is the object, not a copy or reference. In turn the object passed to the callable is consumed and should NOT be used again.

Wraps a task in a TaskWrapper object.

Template Parameters
FxnTypeThe type of the callable that *this will wrap.
ArgsThe types of the arguments that will be forwarded to the callable.

To interact generically with a task, ParallelZone requires the task to be wrapped in a TaskWrapper object. When a TaskWrapper object executes its task the results of the task, if any, are type-erased too. To restore their types TaskWrapper objects can generate unwrapping callables. While TaskWrapper objects and unwrapping callables can be made directly through the TaskWrapper API, the resulting calls are template-heavy. This free function relies on a little bit of under-the-hood template meta-programming to take care of the templating for the caller, thus providing a nicer API.

Parameters
[in]fxnThe callable to wrap.
[in]argsThe arguments to forward to fxn.
Returns
The task and a callable for unwrapping the result of the task.
Exceptions
???This function may throw if capturing the arguments throws. This should be a very unlikely occurrence. Strong throw guarantee.
